# Security Policy

## Repository posture

This repository ingests public earthquake data from public USGS sources and publishes derived public datasets.

Current security posture:

- no credentials should be committed to the repository
- no personal data should be ingested or published
- only derived public exports should be committed
- local scratch files and one-off test exports should remain out of version control

## Reporting

If you find a security issue in the automation, workflow permissions, or published outputs, please report it privately through GitHub Security Advisories if enabled for the repository. If private reporting is not available, avoid posting exploit details in a public issue until the workflow permissions and repository settings have been reviewed.
